# rspress-plugin-typesense <SourceCode href="https://github.com/typesense/rspress-plugin-typesense" />

import { SourceCode } from '@rspress/core/theme';

该插件使用 [Typesense](https://typesense.org/)（一个开源、支持容错拼写的搜索引擎）替换 Rspress 内置搜索。

插件会在构建过程中 (`rspress build`) **自动为文档建立索引**，并提供高度优化的搜索体验，同时支持 Rspress 的多版本与国际化功能。

:::note
这是一个由社区维护的集成方案。
:::

## 安装

import { PackageManagerTabs } from '@rspress/core/theme';

<PackageManagerTabs command="add rspress-plugin-typesense -D" />

## 使用方式

### 1. 启动 Typesense 服务

你可以自行托管 Typesense 服务，也可以使用其云服务。请参考其[快速开始指南](https://typesense.org/docs/guide/install-typesense.html)来完成服务部署，并获取 API Key 与服务地址。

### 2. 配置插件

首先，在你的 `rspress.config.ts` 中添加插件。你必须提供 Typesense 服务信息，以及一个具有**写权限**的 API Key，以便插件能够在构建期间创建 collection 并为文档建立索引。

```ts
// rspress.config.ts
import { defineConfig } from '@rspress/core';
import { pluginTypesense } from 'rspress-plugin-typesense';

export default defineConfig({
  plugins: [
    pluginTypesense({
      collectionName: 'my_docs',
      serverConfig: {
        nodes: [{ url: 'YOUR_TYPESENSE_SERVER_URL' }],
        apiKey: 'YOUR_TYPESENSE_ADMIN_API_KEY', // 需要写权限
      },
    }),
  ],
});
```

### 3. 覆盖搜索组件

接下来，通过 [Custom Theme](/guide/basic/custom-theme) 覆盖 Rspress 默认的 `Search` 组件。

这里请提供一个**仅搜索权限（Search-Only）**的 API Key。出于安全原因，**不要在前端暴露 Admin API Key**。

```tsx
// theme/index.tsx
import { Search as PluginTypesenseSearch } from 'rspress-plugin-typesense/runtime';

const Search = () => {
  return (
    <PluginTypesenseSearch
      docSearchProps={{
        typesenseServerConfig: {
          nodes: [{ url: 'YOUR_TYPESENSE_SERVER_URL' }],
          apiKey: 'YOUR_TYPESENSE_SEARCH_ONLY_API_KEY', // 可安全用于浏览器
        },
      }}
    />
  );
};

export { Search };
export * from '@rspress/core/theme-original';
```

### 4. 构建并建立索引

运行构建命令以生成站点，并将内容索引到 Typesense 中。

<PackageManagerTabs command="run build" />

大功告成！你已经成功为文档集成了支持容错拼写的搜索功能。插件会根据用户当前的语言与版本自动处理搜索过滤。

## 插件配置（后端）

`pluginTypesense` 函数接受如下配置项:

```ts
export interface TypesensePluginOptions {
  /**
   * Typesense 服务连接配置。
   * API Key 必须具有写权限，用于创建 collection 并建立索引。
   */
  serverConfig: ConfigurationOptions;

  /**
   * Typesense collection 的基础名称。
   * 注意: 插件会为不同语言创建独立的本地化 collection（例如 `my_docs_en`）。
   */
  collectionName: string;

  /**
   * 可选的 schema 覆盖配置。
   * 可以是全局配置对象，也可以是按语言区分的配置映射。
   */
  customCollectionSettings?: CustomCollectionSettingsConfig;

  /**
   * 是否将代码块索引到 Typesense 中。
   * 默认为 `false`，以避免搜索噪音和索引体积膨胀。
   */
  indexCodeBlocks?: boolean;

  /**
   * 当索引失败时，是否中断构建流程。
   * 默认为 `true`。
   */
  failOnIndexError?: boolean;

  /**
   * 是否根据当前文档版本自动过滤搜索结果。
   * 默认为 `true`。
   */
  versionedSearch?: boolean;
}
```

### customCollectionSettings

- **类型**:

```ts
type CustomCollectionSettingsConfig =
  | CustomCollectionSettings
  | Record<string, CustomCollectionSettings>;
```

- **默认值**: `undefined`

允许你覆盖 Typesense schema 配置。你可以传入单个全局配置，也可以传入按语言区分的配置映射（例如针对中文或日文自定义 token 分隔符）。

#### 自定义 Schema 与注入自定义数据

对于高级使用场景，例如为分面搜索添加自定义标签，或提升特定页面的搜索排序权重，你可以扩展默认 schema，并在索引前修改记录数据。

操作步骤如下:

1. 在 `customCollectionSettings` 中使用 `getDefaultCollectionFields` 辅助函数，安全地向 collection schema 添加字段。
2. 使用 `transformRecord` 钩子，根据 `route` 填充字段或修改已有权重。

下面的示例展示了如何添加一个用于过滤的自定义 `category` 字段，以及如何提升 "Getting Started" 指南页面的 `page_rank`:

```ts
import {
  pluginTypesense,
  getDefaultCollectionFields,
} from 'rspress-plugin-typesense';

pluginTypesense({
  collectionName: 'my_docs',
  serverConfig: {
    /* ... */
  },

  // 1. 扩展 schema，添加自定义 'category' 字段
  customCollectionSettings: {
    en: {
      fields: (params) => [
        ...getDefaultCollectionFields(params),
        { name: 'category', type: 'string', facet: true, optional: true },
      ],
    },
  },

  // 2. 在索引前修改记录
  transformRecord(record, route) {
    // 示例: 为分面搜索注入自定义标签
    if (route.routePath.startsWith('/api/')) {
      record.category = 'API Reference';
    }

    // 示例: 提升重要页面的搜索优先级
    if (route.routePath.includes('getting-started')) {
      record.weight.page_rank = 100; // 默认值为 0
    }

    return record;
  },
});
```

在前端中，你现在可以向 `<Search />` 组件传入 `typesenseSearchParams: { filter_by: 'category:=API Reference' }`，以限制搜索结果范围。

### indexCodeBlocks

- **类型**: `boolean`
- **默认值**: `false`

默认情况下，插件只会索引标题（`h1-h6`）、段落、列表和表格。启用后，还会提取代码块中的文本内容。

### failOnIndexError

- **类型**: `boolean`
- **默认值**: `true`

默认情况下，如果 Typesense 索引步骤失败（例如网络超时），构建流程会直接失败。如果你希望即使搜索索引失败，CI/CD 部署依然成功，可以将其设置为 `false`。

### versionedSearch

- **类型**: `boolean`
- **默认值**: `true`

如果你的 Rspress 站点使用了多版本功能，插件会为每个被索引的文档添加对应版本标签，并通过虚拟模块将该配置暴露给前端。这可以确保用户只会看到与当前文档版本相关的搜索结果。如果你希望跨所有版本搜索，请将其设置为 `false`。

## 搜索组件 Props（前端）

`<PluginTypesenseSearch />` 组件接受以下属性:

```ts
type SearchProps = {
  docSearchProps: TypesenseDocSearchProps;
  locales?: Locales;
};
```

### `docSearchProps`

- **类型**: `TypesenseDocSearchProps`
- **必填**: 是（必须提供 `typesenseServerConfig`）

会直接传递给底层 `typesense-docsearch-react` 搜索弹窗组件的参数。

:::note
你无需手动提供 `typesenseCollectionName`。插件会通过虚拟模块自动注入 collection 名称。
:::

### `locales`

- **类型**:

```ts
type Locales = Record<
  string,
  { translations: DocSearchProps['translations']; placeholder: string }
>;
```

- **默认值**: `{}`

允许你根据当前语言自定义 placeholder 与搜索弹窗翻译内容。插件内置的翻译列表可查看[这里](https://github.com/typesense/rspress-plugin-typesense/blob/master/src/runtime/locales.ts)。

**示例:**

```tsx
import { Search as PluginTypesenseSearch } from 'rspress-plugin-typesense/runtime';

<PluginTypesenseSearch
  locales={{
    en: {
      placeholder: 'Search documentation',
      translations: {
        button: {
          buttonText: 'Search',
          buttonAriaLabel: 'Search',
        },
      },
    },
    ...ZH_LOCALES,
  }}
/>;
```
