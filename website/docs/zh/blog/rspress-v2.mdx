## 其他

## 在 Markdown/MDX 的编译流程中不再使用 Rust 版本的 MDX 解析器

在使用 lazyCompilation / persistentCache 后，性能的优化效果已经非常显著，我们决定牺牲一些，以换取更好的扩展性和维护性。对比使用 Shiki 等 JS 版本 Markdown 解析辅助工具带来的扩展性和收益，这个权衡是值得的。

这不意味着我们放弃 Markdown 相关生态的 Rust 化，未来我们可能会在 Rslint / Rspack 中继续推进 Rust 版本的 Markdown 解析器的集成。
