# Tailwind CSS

[Tailwind CSS](https://tailwindcss.com/) 是一个以 utility class 为核心的 CSS 框架，用于快速构建自定义用户界面。

Rspress 基于 [Rsbuild](https://rsbuild.rs/) 构建，Tailwind CSS 的配置方式与 Rsbuild 一致。更多详情请参考：

- [Rsbuild - Tailwind CSS v4](https://rsbuild.rs/zh/guide/styling/tailwindcss)
- [Rsbuild - Tailwind CSS v3](https://rsbuild.rs/zh/guide/styling/tailwindcss-v3)

下面介绍如何在 Rspress 中集成 Tailwind CSS v4。

:::tip
Rspress v2 的所有内置组件均采用 BEM 命名规范，不会与 Tailwind CSS 类名冲突。你可以自由使用 Tailwind v4 或 v3，无需担心样式冲突问题。
:::

## 安装依赖

import { PackageManagerTabs } from '@theme';

<PackageManagerTabs command="add tailwindcss @tailwindcss/postcss -D" />

## 创建 PostCSS 配置

在项目根目录创建 `postcss.config.mjs` 文件：

```js title="postcss.config.mjs"
export default {
  plugins: {
    '@tailwindcss/postcss': {},
  },
};
```

## 创建 Tailwind CSS 文件

在项目根目录创建 `tailwind.css` 文件：

```css title="tailwind.css"
@import 'tailwindcss';
```

:::warning
Tailwind CSS v4 使用了 [Cascade Layers](https://developer.mozilla.org/en-US/docs/Learn_web_development/Core/Styling_basics/Cascade_layers) 等现代 CSS 特性，使用前请确认[浏览器兼容性](https://tailwindcss.com/docs/compatibility)。

Tailwind CSS v4 不能与 Sass、Less 或 Stylus 一起使用，请将 Tailwind CSS 的导入保留在纯 `.css` 文件中。
:::

## 配置 Rspress

在 `rspress.config.ts` 中，使用 `globalStyles` 选项引入 Tailwind CSS 文件：

```ts title="rspress.config.ts"
import * as path from 'node:path';
import { defineConfig } from '@rspress/core';

export default defineConfig({
  root: path.join(__dirname, 'doc'),
  globalStyles: path.join(__dirname, 'tailwind.css'),
});
```

## 使用

现在你可以在 MDX 文件中使用 Tailwind 工具类：

```mdx title="doc/index.mdx"
<div className="text-3xl font-bold underline">Hello world!</div>
```
