# Internationalization

To internationalize a Rspress documentation site, complete the following steps:

1. Define I18n text data.
1. Configure the language list, `locales` in `rspress.config.ts`.
1. Configure the default language, `lang` in `rspress.config.ts`.
1. Create docs for each language.
1. Configure sidebar and navbar.
1. Use `useI18n` in custom components.

## Configure `locales`

In `rspress.config.ts`, `locales` configures site-level information such as `lang`, `title`, and `description` for each language.

```ts title="rspress.config.ts"
import { defineConfig } from '@rspress/core';

export default defineConfig({
  // locales is an array of objects
  locales: [
    {
      lang: 'en',
      // The label in the navbar language switcher
      label: 'English',
      title: 'Rspress',
      description: 'Static Site Generator',
    },
    {
      lang: 'zh',
      label: 'з®ЂдЅ“дё­ж–‡',
      title: 'Rspress',
      description: 'йќ™жЂЃзЅ‘з«™з”џж€ђе™Ё',

```

:::tip Note

`themeConfig.locales` also contains all fields from `locales`, but it will be removed in the future. Use `locales` instead.

:::

For other internationalized theme options, see [API type](/api/config/config-theme#locales).

## Configure `lang` default language

After configuring `locales`, set the default site language with [lang](/api/config/config-basic#lang):

```ts title="rspress.config.ts"
import { defineConfig } from '@rspress/core';

export default defineConfig({
  lang: 'en',
});
```

This is important because **Rspress removes the language prefix from routes in the default language**. For example, `/en/guide/getting-started` becomes `/guide/getting-started`.

## Configure \_nav.json and \_meta.json

Use `_nav.json` and `_meta.json` to configure the navbar and sidebar. For details, see [Autogenerated navigation](/guide/basic/auto-nav-sidebar).

### Navigation bar \_nav.json

In the navbar-level `_nav.json`, `text` can be an i18n key. For example:

```json title="_nav.json"
[
  {
    "text": "guide",
    "link": "/guide/start/introduction"
  },
  {
    "text": "api",
    "link": "/api/"
  }
]
```

Here, `text` is `guide`. Rspress translates this value to `жЊ‡еЌ—` or `Guide` based on `i18n.json` and the current language.

### Sidebar \_meta.json

In the sidebar-level `_meta.json`, `label` can be an i18n key. For example:

```json title="_meta.json"
[
  {
    "type": "dir",
    "name": "start",
    "label": "gettingStarted"
  }
]
```

Here, `label` is `gettingStarted`. Rspress translates this value to `ејЂе§‹` or `Getting Started` based on the current language.

## Use `useI18n` in custom components

When writing MDX or developing a [custom theme](/guide/basic/custom-theme), custom components may also need localized text. Use `useI18n` to read it:

import UseI18n from '../../fragments/useI18n.mdx';

<UseI18n />
