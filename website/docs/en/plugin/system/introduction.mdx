# Introduction

The plugin system is a core part of Rspress. It lets you extend Rspress during the site build process. First, let's look at the overall Rspress architecture.

The overall architecture of Rspress is shown in the figure below:

![Rspress Architecture](https://assets.rspack.rs/rspress/assets/rspress-architecture.png)

Rspress has two main parts: **Node Side** and **Browser Runtime**. The plugin system lets you extend both parts. Specifically, plugins can:

- Extend [**Markdown/MDX compilation**](/plugin/system/plugin-api#markdown) by adding [`remark`](https://github.com/remarkjs/remark) or [`rehype`](https://github.com/rehypejs/rehype) plugins.
- [**Add custom pages**](/plugin/system/plugin-api#addpages) on top of Rspress's conventional routing, such as a `/blog` route that renders a custom blog list.
- [**Customize build tool behavior**](/plugin/system/plugin-api#builderconfig) by modifying the underlying [Rsbuild](https://rsbuild.rs) config or adding Rspack/Rsbuild plugins.
- [**Extend page metadata**](/plugin/system/plugin-api#extendpagedata). Rspress calculates metadata such as `title` and `description` for each page. Plugins can extend that logic, and theme code can read the result with [usePageData](/ui/hooks/use-page-data).
- Run [**custom logic**](/plugin/system/plugin-api#beforebuildafterbuild) before and after builds, such as closing event listeners after a build ends.
- [**Add global components**](/plugin/system/plugin-api#globaluicomponents). Since Rspress renders with React, plugins can add global React components such as a BackToTop component or global side-effect component.
