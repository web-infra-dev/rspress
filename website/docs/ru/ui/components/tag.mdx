# Tag

Компонент Tag используется для отображения тегов в **боковой панели или панели навигации** для информации UI. Он поддерживает несколько форматов, включая обычные теги, SVG-иконки, изображения и простой текст.

## Использование

Добавляйте через [метаданные](../../guide/use-mdx/frontmatter.mdx), и тег будет отображаться в боковой панели или панели навигации:

```mdx title="foo.mdx"
---
tag: новинка
---

# Foo

какой-нибудь текст
```

<Tag tag="новинка" />

Поддерживается и вывод нескольких тегов:

```mdx
---
tag: новинка, экспериментально
---
```

<div style={{ display: 'flex', gap: '8px' }}>
  <Tag tag="новинка, экспериментально" />
</div>

Также можно указывать теги через файл `_meta.json`:

```json title="_meta.json"
{
  "name": "foo",
  "type": "file",
  "tag": "новинка, экспериментально"
}
```

:::tip Совет

Компонент Tag специально предназначен для отображения в боковой панели или панели навигации. При использовании в контенте MDX, пожалуйста, используйте вместо него [компонент Badge](./badge.mdx).

:::

## Типы тегов

Чтобы упростить передачу значений в блок метаданных, компонент Tag поддерживает несколько типов тегов:

### Общие теги

Некоторые распространённые теги встроены и будут отображаться с помощью `<Badge />`. Все распространённые теги перечислены ниже:

| Тег            | Соответствующий UI Badge       |
| -------------- | ------------------------------ |
| `tip`          | <Tag tag="совет" />            |
| `info`         | <Tag tag="информация" />       |
| `warning`      | <Tag tag="предупреждение" />   |
| `danger`       | <Tag tag="опасность" />        |
| `new`          | <Tag tag="новинка" />          |
| `experimental` | <Tag tag="экспериментально" /> |
| `deprecated`   | <Tag tag="устарело" />         |

:::tip Совет

Компонент Tag является `ejectable`. Когда вы хотите расширить стандартные теги, вы можете использовать оборачивание или извлечение для модификации компонента Tag.

Ниже приведён пример того, как этот сайт использует оборачивание для добавления пользовательского тега `ejectable`:

```tsx title="theme/components/Tag/index.tsx"
import {
  Badge as BasicBadge,
  Tag as BasicTag,
} from '@rspress/core/theme-original';

export const Tag = ({ tag }: { tag: string }) => {
  if (tag === 'ejectable') {
    return <BasicBadge text="ejectable" type="tip" />;
  }
  return <BasicTag tag={tag} />;
};
```

```mdx title="index.mdx"
---
tag: новинка, ejectable
---
```

:::

### Строка SVG

Строку SVG можно передать напрямую:

```mdx
---
tag: <svg xmlns="http://www.w3.org/2000/svg" width="20" height="20" viewBox="0 0 24 24"><path fill="currentColor" d="M12 2L2 7l10 5 10-5-10-5z"/></svg>
---
```

### URL-адрес изображения

Вы можете использовать внешний URL или data URL в качестве тега:

```mdx
---
tag: https://example.com/icon.png
---
```

```mdx
---
tag: data:image/svg+xml;base64,...
---
```

### Обычный текст

Любой текст, не соответствующий указанным выше шаблонам, будет отображён в виде простого текстового Badge:

```tsx
import { Tag } from '@theme';

<Tag tag="v1.0.0" />
<Tag tag="Beta" />
```

<div style={{ display: 'flex', gap: '8px' }}>
  <Tag tag="v1.0.0" />
  <Tag tag="Beta" />
</div>

## Пропсы

```ts
interface TagProps {
  /**
   * Содержимое тега. Поддерживает:
   * - Общие теги: 'tip', 'info', 'warning', 'danger', 'new', 'experimental', 'deprecated'
   * - Несколько тегов вместе: 'new, experimental'
   * - SVG-строку: '<svg>...</svg>'
   * - URL изображения: 'https://...' или 'data:...'
   * - Простой текст: любая другая строка
   */
  tag?: string;
}
```
