# llms.txt (SSG-MD)

## Что такое SSG-MD?

Rspress предоставляет экспериментальную возможность Static Site Generation to Markdown (SSG-MD — генерация статического сайта в Markdown) — это совершенно новая функция. Аналогично процессу [генерации статического сайта](./ssg), SSG-MD рендерит ваши страницы в виде файлов Markdown вместо HTML-файлов и генерирует [`llms.txt`](#llms-txt) и llms-full.txt. Это упрощает большим языковым моделям понимание и использование вашей технической документации.

Чтобы лучше понять SSG-MD, вот аналогия между SSG и SSG-MD:

| Сравнение                     | SSG                                        | SSG-MD                                     |
| ----------------------------- | ------------------------------------------ | ------------------------------------------ |
| **Полное название**           | Генерация статического сайта               | Генерация статического сайта в Markdown    |
| **Цель оптимизации**          | SEO (оптимизация для поисковых систем)     | GEO (оптимизация для генеративных движков) |
| **Ориентирован на**           | Поисковые краулеры                         | LLM / системы векторного поиска            |
| **Файл-индекс**               | [`sitemap.xml`](https://www.sitemaps.org/) | [`llms.txt`](https://llmstxt.org/)         |
| **Файл с полным контентом**   | -                                          | `llms-full.txt`                            |
| **Ключевой метод рендеринга** | `renderToString`                           | `renderToMarkdownString`                   |
| **Способ доступа**            | `/guide/start/introduction.html`           | `/guide/start/introduction.md`             |

## Что такое llms.txt? {#llms-txt}

[`llms.txt`](https://llmstxt.org/) — это новый стандартный файл, размещаемый в корне сайта, чтобы помочь большим языковым моделям лучше понимать и использовать содержимое сайта.

Поскольку у LLM ограничено контекстное окно и они не могут обработать весь HTML-код сайта, а преобразование сложного HTML (с навигацией, рекламой, JavaScript) в чистый текст является сложным и неточным процессом, `llms.txt` предоставляет структурированный индекс сайта в формате Markdown: URL страниц и описание их содержимого. Это позволяет ИИ быстро находить и понимать ключевую информацию.

Простыми словами:

- `sitemap.xml` → «карта сайта» для поисковых систем
- `llms.txt` → «оглавление документации» для ИИ

Пример структуры вывода:

```tree
doc_build
├── llms.txt
├── llms-full.txt
├── guide
│   └── start
│       └── introduction.md
└── ...
```

Пример содержимого `llms.txt`:

```txt
# Rspress

> Rspress is a static site generator based on Rspack.

## Docs

- [Introduction](/guide/start/introduction.md): Introduction to Rspress
- [Quick Start](/guide/start/quick-start.md): Quick Start
```

## Почему SSG-MD?

В фронтенд-фреймворках на базе React с динамическим рендерингом часто возникает проблема сложности извлечения статической информации. Эта проблема существует и в MDX, где файлы `.mdx` содержат как Markdown-контент, так и поддержку встраивания React-компонентов, что повышает интерактивность документов. В Rspress пользователи могут использовать [фрагменты MDX](../use-mdx/components.mdx), кастомные компоненты, хуки React, файлы tsx в качестве маршрутов и т. д., чтобы усилить выразительность содержимого документов. Однако такие динамические элементы сложно преобразовать в формат Markdown, и даже если преобразовать HTML, сгенерированный на этапе SSG, в Markdown, результаты часто оказываются неудовлетворительными.

[Генерация статического сайта (SSG)](./ssg) позволяет генерировать статические HTML-файлы для краулеров поисковых систем, улучшая [SEO](https://ru.wikipedia.org/wiki/%D0%9F%D0%BE%D0%B8%D1%81%D0%BA%D0%BE%D0%B2%D0%B0%D1%8F_%D0%BE%D0%BF%D1%82%D0%B8%D0%BC%D0%B8%D0%B7%D0%B0%D1%86%D0%B8%D1%8F). SSG-MD решает аналогичные проблемы, улучшая [GEO](https://ru.wikipedia.org/wiki/Generative_Engine_Optimization) и качество статической информации для больших языковых моделей. По сравнению с преобразованием HTML в Markdown, виртуальный DOM React во время рендеринга предоставляет более качественный источник информации.

## Как реализован SSG-MD?

1. Внутри Rspress реализован метод `renderToMarkdownString`, аналогичный `renderToString` из `react-dom`, который рендерит React-компоненты в строки Markdown:

```tsx
import { renderToMarkdownString } from 'react-render-to-markdown';

// Элементы HTML конвертируются в соответствующий синтаксис Markdown
renderToMarkdownString(
  <div>
    <strong>foo</strong>
    <span>bar</span>
  </div>,
);
// Вывод: '**foo**bar'

// Поддерживает компоненты и хуки React
const Article = () => {
  return (
    <>
      <h1>Привет, мир</h1>
      <p>Это параграф.</p>
    </>
  );
};
renderToMarkdownString(<Article />);
// Вывод: '# Привет, мир\n\nЭто параграф.\n'
```

В принципе, этот API работает для любого сайта, построенного на React; подробнее смотрите [react-render-to-markdown](https://www.npmjs.com/package/react-render-to-markdown), если интересно.

2. Предоставляет переменную окружения `import.meta.env.SSG_MD`, которая позволяет пользователям легко различать рендеринг в режиме SSG-MD и рендеринг в браузере внутри React-компонентов, тем самым обеспечивая более гибкую настройку содержимого:

```tsx
export function Tab({ label }: { label: string }) {
  if (import.meta.env.SSG_MD) {
    return <>{`** Это вкладка с именем ${label}**`}</>;
  }
  return <div>{label}</div>;
}
```

3. Внутренняя библиотека компонентов Rspress была адаптирована для SSG-MD, чтобы обеспечить рендеринг разумного Markdown-контента на этапе SSG-MD. Например:

```tsx
<PackageManagerTabs command="create rspress@latest" />
```

Будет отрендерено как:

````md
```sh [npm]
npm create rspress@latest
```

```sh [yarn]
yarn create rspress
```

```sh [pnpm]
pnpm create rspress@latest
```

```sh [bun]
bun create rspress@latest
```

```sh [deno]
deno init --npm rspress@latest
```
````

## Возможности

- Рендерит каждую страницу сайта в виде файла `.md`, что удобно для векторизации или передачи большим языковым моделям. Доступ к `/guide/start/introduction.html` можно получить, заменив суффикс `.html` на `.md`.
- Генерирует [`llms.txt`](https://llmstxt.org/), в котором отображаются заголовок и описание каждой страницы в порядке навигации и боковой панели.
- Генерирует `llms-full.txt`, содержащий Markdown-контент каждой страницы, что удобно для пакетного импорта.
- Поддерживает многоязычные сайты: для языков, отличных от языка по умолчанию, выводятся соответствующие `{lang}/llms.txt` и `{lang}/llms-full.txt`.

## Пример вывода

```tree
doc_build
├── llms.txt
├── llms-full.txt
├── guide
│   └── start
│       └── introduction.md
└── ...
```

Фактические файлы размещаются в каталоге сборки (например, `guide/start/introduction.md`), а `url` в `llms-full.txt` будет содержать префикс сайта, например `/guide/start/introduction.md`.

Пример фрагмента `llms-full.txt`:

```md
---
url: /guide/start/introduction.md
---

# Introduction

...
```

## Как включить

Включите опцию `llms` в файле `rspress.config.ts`, чтобы генерировать указанные выше файлы на этапе сборки:

```ts title="rspress.config.ts"
import { defineConfig } from '@rspress/core';

export default defineConfig({
  llms: true,
});
```

После выполнения команды `rspress build` в выходной директории (по умолчанию `doc_build`) вы сможете увидеть файлы `llms.txt`, `llms-full.txt`, а также файлы `.md`, соответствующие каждому маршруту.

:::warning Предупреждение

`llms` — это экспериментальная возможность, предназначенная в первую очередь для генерации Markdown-данных, удобных для использования большими языковыми моделями или системами поиска. В будущих версиях она будет постоянно улучшаться и может иметь проблемы со стабильностью или совместимостью.

Если функции SSG и SSG-MD невозможно включить из-за несовместимости кода с SSR, пожалуйста, используйте [@rspress/plugin-llms](/plugin/official-plugins/llms).

:::

## Настройка разбиения MDX (опционально)

Когда в документах используются пользовательские компоненты, вы можете управлять тем, какие компоненты следует сохранить, а какие преобразовать в обычный текст при конвертации в Markdown, с помощью опции `remarkSplitMdxOptions`:

```ts title="rspress.config.ts"
import { defineConfig } from '@rspress/core';

export default defineConfig({
  llms: {
    remarkSplitMdxOptions: {
      excludes: [[['Demo'], '@project/components']],
    },
  },
});
```

- `excludes`: Соответствующие компоненты будут преобразованы в обычный текст, с наивысшим приоритетом.
- `includes`: Если задано, то только соответствующие компоненты будут сохранены, а остальные преобразованы в обычный текст.
- При одновременной настройке сначала применяется `excludes`, затем фильтрация по `includes`.
